# Spring Cloud架构方案

**配置中心**：默认采用Spring Cloud Config，亦可使用Spring Cloud Consul(流行)、Spring Cloud Alibaba Nacos(流行)代替。

**服务发现**：默认采用Netflix Eureka，亦可使用Spring Cloud Consul(流行)、Spring Cloud ZooKeeper、Etcd等代替。

**服务网关**：默认采用Netflix Zuul，亦可使用Spring Cloud Gateway(流行)代替。

**服务治理**：默认采用Netflix Hystrix，亦可使用Sentinel(流行)、Resilience4j代替。

**进程内负载均衡**：默认采用Netfilix Ribbon，亦可使用Spring Cloud Loadbalancer(流行)代替。

**声明式 HTTP 客户端**：默认采用Spring Cloud OpenFeign。声明式的 HTTP 客户端其实没有找替代品的必要性，如果需要，可考虑Retrofit，或者使用 RestTemplete 乃至于更底层的OkHTTP、HTTPClient以命令式编程来访问，多写一些代码而已了。
